# Индивидуальная экосистема продаж и автоматизации: Частный мастер террариумов (Москва, 2026)

Репозиторий содержит готовую инфраструктуру («бизнес в коробке») для частного специалиста по изготовлению террариумов на заказ в Москве и Московской области.

## 🗂 Структура репозитория

```
├── README.md                          # Главный файл с описанием проекта
├── .env.example                       # Шаблон всех переменных окружения
├── bot/                               # Telegram-бот (прием заявок, уведомления, мини-CRM)
│   ├── main.py                        # Исходный код бота (Python / aiogram)
│   ├── requirements.txt               # Зависимости Python
│   ├── spec-telegram-bot.md           # Техническая спецификация бота
│   └── deploy-railway.md              # Инструкция по деплою на Railway
├── crm/                               # База данных и CRM-системы
│   ├── data-model.md                  # Описание структуры данных (Сущности, Воронка)
│   ├── schema.sql                     # SQL-схема для PostgreSQL (Railway)
│   ├── google-sheets-structure.md     # Структура колонок для Google Sheets / Airtable
│   ├── integration-flow.md            # Схема и описание потока интеграции (Landing -> CRM -> Bot)
│   └── api-handler-example.ts         # Пример бэкенд-обработчика для приема лидов
├── docs/                              # Стратегия, маркетинговые тексты и инструкции
│   ├── market-2026-summary.md         # Анализ рынка Москвы 2026
│   ├── positioning-individual.md      # УТП и позиционирование частного мастера
│   ├── channel-strategy-individual.md # Стратегия продвижения (Авито, Профи, Яндекс, Соцсети)
│   ├── copy-avito-profile-listings.md # Готовые тексты для профиля и объявлений Авито
│   ├── copy-profi-profile.md          # Готовые тексты для Профи.ру
│   ├── copy-yandex-services-card.md   # Готовые тексты для Яндекс Услуг / Яндекс Бизнеса
│   ├── copy-social-bios.md            # Био и описания для VK, Telegram и соцсетей
│   ├── lead-workflow-individual.md    # Алгоритм работы мастера с заявками (от и до)
│   └── message-templates-ru.md        # Шаблоны сообщений для общения с клиентами
└── landing/                           # Одностраничный лендинг (Next.js / React / Tailwind)
    ├── README.md                      # Инструкция по запуску и деплою лендинга
    ├── package.json                   # Зависимости Node.js
    ├── content-landing-ru.md          # Текстовое наполнение всех блоков лендинга
    ├── components/
    │   └── ContactForm.tsx            # Компонент формы обратной связи
    └── pages/
        ├── index.tsx                  # Главная страница лендинга
        └── api/
            └── contact.ts             # API-роут для отправки лидов в CRM/Telegram
```

## 🚀 Целевая архитектура

1. **Vercel**: Хостинг Next.js лендинга (`/landing`). Максимальная скорость загрузки, бесплатный тариф, автоматический SSL.
2. **Railway**: Хостинг Telegram-бота (`/bot`) и базы данных PostgreSQL (`/crm/schema.sql`). Простой деплой из GitHub, минимальные затраты.
3. **Google Sheets / Airtable**: Легкая бесплатная CRM-система для ручного ведения заказов и клиентов.
4. **Telegram**: Основной хаб мастера. Получение мгновенных уведомлений о лидах, быстрое изменение статусов заказов через команды бота.
