# Модель данных и сущности CRM (data-model.md)

CRM-система частного мастера террариумов в Москве спроектирована максимально прагматично. Она состоит из 4 ключевых сущностей: **Lead** (Входящая заявка), **Client** (Заказчик), **Order** (Заказ) и **Status** (Этапы производственной воронки).

---

## 1. Сущность: Lead (Заявка)
Фиксирует все первичные обращения из любого канала до того, как клиенту выставлена итоговая смета и внесена предоплата. Совпадает со структурой SQL-таблицы `leads`.

| Поле SQL | Тип данных | Описание / Пример |
| :--- | :--- | :--- |
| `lead_id` | UUID | Уникальный идентификатор заявки |
| `source` | String | Источник (`Avito`, `Profi`, `Yandex`, `Landing`, `Telegram`) |
| `client_name` | String | Имя клиента (`Алексей`) |
| `contact_info` | String | Номер телефона или ник в мессенджере (`+79991112233`) |
| `preferred_messenger` | String | Способ связи (`telegram`, `whatsapp`, `phone`) |
| `pet_type` | String | Для какого питомца (`Бородатая агама`) |
| `dimensions` | String | Желаемые габариты (`80х40х40 см`) |
| `comment` | Text | Пожелания (`Нужен 3D-фон под красную скалу`) |
| `status` | String | Текущий этап (`NEW_LEAD`, `ESTIMATING`, `IN_PRODUCTION`, `COMPLETED`) |
| `is_converted` | Boolean | Переведен ли в реальный заказ (`true` / `false`) |
| `created_at` | Timestamp | Время поступления заявки (`2026-06-18 14:30:00`) |

---

## 2. Сущность: Client (Клиент)
Создается автоматически при переводе Лида в Заказ или при добавлении постоянного заказчика (`clients`).

| Поле SQL | Тип данных | Описание / Пример |
| :--- | :--- | :--- |
| `client_id` | UUID | Уникальный идентификатор клиента |
| `full_name` | String | ФИО клиента (`Смирнов Алексей Викторович`) |
| `contact_info` | String | Единый контакт (телефон / Telegram) |
| `delivery_address`| Text | Адрес доставки (`г. Москва, ул. Ленина, д. 5, кв. 12`) |
| `notes` | Text | Особые отметки (`Просил звонить только после 18:00`) |
| `total_orders` | Integer | Количество выполненных заказов (`2`) |
| `total_revenue` | Decimal | Общая сумма принесенной выручки (`34500.00`) |
| `created_at` | Timestamp | Дата регистрации клиента |

---

## 3. Сущность: Order (Заказ)
Основная производственная карточка, с которой мастер работает каждый день (`orders`).

| Поле SQL | Тип данных | Описание / Пример |
| :--- | :--- | :--- |
| `order_id` | UUID | Уникальный системный номер |
| `order_number` | String | Читаемый номер заказа (`ORD-2026-0042`) |
| `client_id` | Foreign Key | Ссылка на таблицу `clients` |
| `status` | String | Текущий этап воронки (см. раздел 4) |
| `package_name` | String | `Индивидуальный Кастом`, `Комплекс Под Ключ`, `Палюдариум` |
| `dimensions` | String | Итоговые согласованные габариты (`80х40х40`) |
| `glass_thickness` | Integer | Толщина стекла в мм (`5`) |
| `has_3d_background`| Boolean | Наличие авторского 3D-фона (`true`) |
| `has_equipment` | Boolean | Наличие УФ-света и обогрева (`true`) |
| `total_price` | Decimal | Общая стоимость заказа (`14500.00`) |
| `prepayment` | Decimal | Внесенный аванс (`7000.00`) |
| `deadline` | Date | Дедлайн сдачи клиенту (`2026-06-25`) |
| `notes` | Text | Технические заметки по склейке |
| `created_at` | Timestamp | Дата создания заказа |

---

## 4. Этапы производственной воронки (Status)

В системе заложены 6 четких производственных статусов для карточек `leads` и `orders`:

1. 🟡 `NEW_LEAD` — Новая необработанная заявка.
2. 🟠 `ESTIMATING` — Идет согласование чертежа, оборудования и сметы.
3. 🔵 `PREPAYMENT_RECEIVED` — Аванс 50% получен, стекло закупается / нарезается на станке.
4. 🟣 `IN_PRODUCTION` — Корпус склеен, идет процесс лепки и сушки авторского 3D-фона.
5. 🟢 `READY_FOR_DELIVERY` — Террариум полностью готов, отснят видеоотчет, ожидается отгрузка.
6. 🏁 `COMPLETED` — Террариум передан клиенту, доплата 50% получена, сделка закрыта.
* 🔴 `CANCELLED` — Отказ / Отмена.
