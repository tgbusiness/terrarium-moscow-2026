# Поток интеграции данных: Landing → CRM → Telegram Bot

Этот файл описывает техническую архитектуру передачи данных, обеспечивающую мгновенное оповещение мастера о новых заявках и синхронизацию с CRM.

```
[ Клиент заполняет ContactForm ]
               │
               ▼
[ POST-запрос на /api/contact (Next.js / Railway) ]
               │
       ┌───────┴────────────────────────┐
       ▼                                ▼
[ Запись строки в Google Sheet ]  [ Вызов Telegram API (sendMessage) ]
 (или INSERT в PostgreSQL)              │
                                        ▼
                         [ Уведомление в Telegram мастера ]
                           "🔥 НОВЫЙ ЛИД: Алексей..."
                                        │
                                        ▼
                         [ Мастер нажимает In-line кнопку ]
                           «Взять в работу / Обновить статус»
```

## Пошаговый сценарий под капотом:

1. **Генерация лида:** Клиент заходит на лендинг (`/landing`), выбирает нужные габариты или пакет и заполняет форму (Имя, Телефон, Питомец).
2. **Отправка POST-запроса:** Компонент React `ContactForm.tsx` делает асинхронный POST-запрос с JSON-телом на API-роут `/api/contact`.
3. **Обработка на сервере:** API-обработчик (`api-handler-example.ts`) выполняет параллельно две задачи:
   - **Задача А (Уведомление):** Делает HTTP-запрос к API Telegram (`https://api.telegram.org/bot<TOKEN>/sendMessage`) и отправляет красиво отформатированное HTML-сообщение в личку мастера (или в секретный рабочий чат).
   - **Задача Б (Сохранение в CRM):** Подключается к Google Sheets API по сервисному ключу и дописывает новую строку на вкладку `Leads & Orders` (либо выполняет `INSERT INTO leads` в PostgreSQL).
4. **Управление через Бот:** Мастер видит уведомление в Telegram. Прямо под сообщением находятся in-line кнопки: `[Взять в работу]`, `[Выставить смету]`, `[Отказ]`. При нажатии на кнопку бот автоматически меняет статус в Google Таблице или Базе данных!
